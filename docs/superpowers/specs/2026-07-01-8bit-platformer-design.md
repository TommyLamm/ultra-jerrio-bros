# 8-bit 橫向平台遊戲設計規格

日期：2026-07-01

## 目標

製作一個可直接用瀏覽器開啟的 2D 橫向捲軸平台遊戲。遊戲要盡量還原經典 8-bit 平台遊戲的手感與第一關節奏：跑跳、慣性、踩怪、頂磚、問號磚、金幣、成長道具、倒數時間、分數、生命、旗桿終點與重新開始。

素材與角色必須是原創像素風致敬，不直接複製任天堂或 Super Mario Bros. 的受保護角色、美術、音效、地圖配置或商標。還原重點放在玩法語言、物理節奏、畫面密度與 8-bit 氛圍。

## 使用者範圍

玩家只需要打開 `index.html` 就能遊玩，不需要安裝套件或執行 build step。遊戲以桌面鍵盤為主要輸入方式，支援：

- 方向鍵移動，`Z` 跳躍，`X` 跑步；取得投射道具後 `X` 也會射擊。
- `A/D` 移動，`Space` 跳躍，`Shift` 跑步；取得投射道具後 `Shift` 也會射擊。
- `Enter` 開始、重開或從結算畫面回到遊戲。

## 推薦方案

採用單檔 Canvas 架構。`index.html` 內包含 HTML、CSS 與 JavaScript，使用 `<canvas>` 繪製遊戲。這符合「直接打開就能玩」的需求，也能把所有邏輯封裝在一個可攜檔案中。

不使用 Phaser 或其他外部遊戲框架，因為本需求的重點是單檔交付與可控手感。碰撞、相機、物理與關卡資料會以小型內部模組區塊組織，避免單純堆疊事件處理。

## 遊戲結構

`index.html` 的 JavaScript 會分成以下區塊：

- `Constants`：畫面尺寸、tile 尺寸、重力、速度、分數與時間設定。
- `Input`：鍵盤狀態、按下瞬間、同時支援兩套鍵位。
- `Audio`：用 Web Audio 產生短促 8-bit 風格音效，包括跳躍、金幣、頂磚、受傷、踩怪與過關。
- `Level`：tile map、物件生成資料、旗桿終點與重生點。
- `Physics`：AABB 碰撞、tile 碰撞、平台站立、垂直與水平分離解算。
- `Entities`：玩家、敵人、道具、粒子、金幣與浮動分數。
- `GameState`：開始、遊玩、死亡、過關、遊戲結束。
- `Renderer`：像素風背景、tiles、sprites、HUD、相機與畫面轉場。

## 核心玩法

玩家從左到右通過一段第一關風格關卡。關卡包含平地、磚塊平台、問號磚、缺口、敵人、管道、階梯與旗桿。玩家能：

- 走路、跑步、跳躍，並透過長短按控制跳躍高度。
- 從下方頂問號磚取得金幣或成長道具。
- 從下方頂普通磚，大型狀態可破壞部分磚塊。
- 踩敵人使其消失並反彈。
- 撞到敵人時受傷；小型狀態死亡，大型狀態退回小型並短暫無敵。
- 吃到成長道具後變大，提升容錯並可破壞部分磚。
- 大型狀態再取得進階道具後進入投射狀態，可發射原創短程彈體擊退敵人。
- 抵達旗桿後進入過關結算，依剩餘時間加分。

## 手感設定

遊戲使用固定時間步更新，避免不同螢幕刷新率造成手感不一致。物理預設：

- 有水平加速度與摩擦，而不是立即滿速或立即停止。
- 跑步鍵提高最高速度與加速度。
- 跳躍初速固定，放開跳鍵會提早截斷上升速度，形成可變跳。
- 落地時允許極短的跳躍緩衝與 coyote time，讓瀏覽器鍵盤操作更穩。
- 相機水平追蹤玩家，但不向左倒退太多，維持經典橫向推進感。

## 視覺設計

視覺採用經典 8-bit 致敬方向：

- 16px tile 基準，Canvas 放大顯示並設定 `image-rendering: pixelated`。
- 高飽和藍天、棕色地面、亮色問號磚、綠色管道、簡潔 HUD。
- 原創主角輪廓：紅色調、帽子感與工作服感可以作為復古平台角色語言，但不能複製 Mario 的具體造型。
- 敵人、道具與磚塊都使用原創像素造型。
- 不使用外部圖片素材；所有圖形用 Canvas 像素繪製函式產生。

## 關卡內容

第一版只做一個完整關卡。長度約 180 到 220 個 tiles，目標遊玩時間約 60 到 120 秒。關卡應包含：

- 起始安全區。
- 第一組敵人教學。
- 問號磚與金幣段落。
- 第一個成長道具。
- 一個大型狀態可取得的進階投射道具。
- 一到兩個管道障礙。
- 小型缺口與平台跳躍。
- 階梯接近終點。
- 旗桿與過關行走收尾。

關卡是原創配置，不重建 SMB 1-1 的精確地圖。

## 失敗與重開

玩家死亡時播放短暫動畫，扣生命並從起點重開。生命歸零後進入 Game Over，按 `Enter` 可重新開始。掉出畫面底部也視為死亡。時間倒數到零會死亡。

## 測試策略

因為是單檔 Canvas 遊戲，測試分為自動化與手動驗收：

- 在 `index.html` 中保留可由瀏覽器 console 或測試腳本呼叫的輕量 `window.__gameTest` API，用於測試碰撞、關卡載入與基本狀態轉換。
- 使用 Playwright 開啟 `index.html`，確認 Canvas 存在、畫面非空白、鍵盤輸入能移動玩家、跳躍後位置改變、死亡或過關狀態可重開。
- 手動驗收物理手感：跑跳、踩怪、頂磚、吃道具、投射攻擊、受傷、旗桿結算。

## 交付標準

完成後應交付：

- `index.html`：可直接開啟的完整遊戲。
- 可玩的第一關風格體驗。
- 無外部依賴。
- 沒有直接複製受保護素材或地圖。
- 已用瀏覽器測試 Canvas 渲染與核心互動。
